Mac连接win8.1 Remote Desktop Connection问题修复

"Remote Desktop Connection cannot verify the identity of the computer that you want to connect to." 不过是windows升级到8.1，Mac端Remote Desktop Connection就没法成功远程windows桌面，连接时提示个这，影响生活啊！

软件和系统版本：

  • windows8.1
  • Yomemite
  • Remote Desktop Connection软件版本2.1.1

用其他的设备比如win7，远程桌面链接我的win8.1是OK的，windows的防火墙已经关掉了，有两种可能：

  1. 我的client端RDC软件出问题了，但是我用这个软件可以直接远程桌面win7的系统
  2. 在防火墙关掉的情况下，win8.1还是把RDC的远程桌面请求给拒绝了，也就是报错信息说的感觉。

找了半天终于翻出别人的一个靠谱解决方法，点击这里查看原帖。

我的windows是中文系统，下面把方法说明一下方便遇到问题的windows中文系统的同学们查看：

  1. 首先要保证防火墙是可以允许RDP连接的。我极端的关掉它这种做法不推荐
  2. 打开控制台，mmc.exe
  3. 点击文件->添加或删除管理单元，在可用的管理单元中选择本地计算机策略后添加；此时看到根目录控制台下出现了本地计算机策略
  4. 点击本地计算机策略->计算机配置->管理模板->windows组件->远程桌面服务->远程桌面会话主机->安全
  5. 这个目录下的所有项初始都是未配置，有两项需要改变：
    • 远程(RDP)连接要求使用指定的安全层，改成已启用
    • 要求使用网络级别的身份验证对远程连接的用户进行身份验证，改成已禁用

打完这套组合拳，重启下远程桌面连接的服务就好了： 右击 计算机->管理->服务和应用程序->服务，找到Remote Desktop Services右击->重新启动

现在Mac用可以用Remote Desktop Connection连接上windows8.1了，丝的使用感受又回来了。
